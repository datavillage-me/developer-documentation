getAuthenticationUrl

getAuthenticationUrl: ((activationCallbackUrl: string, type?: POD_IDENTITY_PROVIDER, issuer?: string) => Promise<string>)

Type declaration

    • (activationCallbackUrl: string, type?: POD_IDENTITY_PROVIDER, issuer?: string): Promise<string>

    • Obtain a URL to the passport endpoint that will start the login flow for the given pod type. The flow will redirect the browser to the appropriate page for the given pod type and issuer, and process the resulting authentication tokens to produce an authentication cookie valid for the passport API. For Solid authentication, is is recommended to provide the identity issue. If not, the passport backend will use a default one, like http://inrupt.net .

      Parameters

      • activationCallbackUrl: string

        where to redirect the browser after the full passport login process

      • Optional type: POD_IDENTITY_PROVIDER

        the type of identity provider

      • Optional issuer: string

        the specific identity provider (mostly for Solid)

      Returns Promise<string>
